我无语了。。。

我明明设置好了 pip 的默认安装位置，并且空间足够大，但是下载到最后的时候，还是会报错：空间不足。ERROR: Could not install packages due to an OSError: [Errno 28] No space left on device

注：哦对，该文章只对于 C 盘空间所剩无几的朋友们有效（说起来都是泪啊）

最后偶然间了解到：
pip 安装下载过程中，还会有一个地方作为临时文件夹，而这个文件夹一般是默认在 c 盘（大无语，真相了）。

所以即使你修改了 “site.py” 文件里面对应的路径，过程中的路径其实你还是没有改动的，还是会默认在 c 盘。而这时候如果你 C 盘空间所剩无几，无法提供充足的空间，那你的下载过程肯定九无法正常了，所以我们要做的就是修改 “temp” 文件夹所在位置

在 windows 系统中使用 pip 安装 Python 包时，如果你想要修改临时文件的存储路径，可以通过设置环境变量 TMPDIR 来实现。

将 C:/path/to/new/temp 替换为你希望 pip 使用的新临时目录路径。然后再下载那个体积比较大的包，就没有问题了

# 虚拟环境清理缓存方法

conda clean 是 Anaconda 中的一个命令，用于清理不再需要的文件和缓存，以释放磁盘空间。这个命令非常有助于管理 Anaconda 环境中的存储使用情况。

conda clean --all 这个命令的作用是删除所有可以安全删除的未使用的文件。具体来说，它执行以下操作：

  1. 清理包缓存：删除下载的包文件，这些文件通常存储在 ~/anaconda3/conda/pkgs/ 或 ~/.conda/pkgs/ 目录下。
  2. 删除旧的包版本：如果一个包的新版本已经安装， conda clean 会删除旧版本的包文件。
  3. 清理索引缓存：删除 conda 的索引缓存，这包括包的元数据和依赖信息。
  4. 删除锁文件：删除在包安装过程中创建的锁文件。
  5. 清理未使用的构建环境：删除在构建包时创建的不再需要的环境。

运行 conda clean --all 会释放大量空间，特别是如果你经常安装和卸载包的话。然而，请注意，这个命令不会删除当前安装的包或环境，也不会影响你的工作目录或项目文件。

要运行这个命令，你需要打开 Anaconda Prompt（在 Windows 上）或终端（在 Linux 或 macOS 上），然后输入：

或者，如果你只想清理特定的未使用文件，可以使用不同的选项，例如：

  • conda clean --tarballs ：只清理下载的包文件。
  • conda clean --index-cache ：只清理索引缓存。

使用 conda clean 是维护 Anaconda 环境和避免磁盘空间不足问题的一个好习惯。
